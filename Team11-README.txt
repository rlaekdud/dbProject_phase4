1. 실행 및 사용 방법
 - 프로그램 실행에 앞서, Java Runtime Environment (JRE)를 설치하고 데이터베이스 연결을 위한 JDBC 드라이버(ojdbc10) 를 준비합니다.
 - JAVA 11 버전 사용
 - ojdbc10 사용
 - Tomcat v8.5사용
 - (Intellij) Project Structure → Modules → Dependencies
  - apache-tomcat-8.5.95\lib\ 안에 있는 servlet-api.jar 파일이랑 jsp-api.jar 파일 추가
  - DB설정은 Knuflix_intellij/src/repository 경로의 Repository.java 파일을 이용합니다.

2. 기능 설명
 1. 회원가입 및 로그인

  - 회원가입 1/3 단계: 회원가입 하고자 하는 아이디와 비밀번호를 입력합니다. 이때, 이미 회원가입된 아이디를    입력할 경우 '중복된 아이디입니다. 다시 입력해주세요.'라는 에러를 출력합니다.

  - 회원가입 2/3 단계: 이름과 전화번호를 입력합니다. 이때, 이미 회원가입된 전화번호를 입력할 경우 '전화번호 입력이 옳지 않습니다. 다시 입력해주세요.'라는 에러를 출력합니다.

 - 회원가입 3/3 단계: 결제정보인 은행명과 계좌번호를 입력합니다. 해당 단계까지 성공적으로 마무리하면, 회원가입이 완료됩니다.

  - 로그인 단계: 이미 계정이 있는 경우 '이미 계정이 있으신가요?'를 선택하여 아이디와 비밀번호를 입력하여 로그인 합니다. 이때 아이디 또는 비밀번호를 잘못 입력할 경우 '잘못된 로그인 정보입니다. 다시 입력해주세요'라는 에러를 출력합니다.

 2. 홈 메뉴

  - 내 정보: 붉은 사람 모양의 아이콘을 클릭하여 접근할 수 있습니다.
    ❍ 내 정보 변경: 기존 비밀번호와 새 비밀번호를 입력하여 비밀번호를 변경할 수 있습니다. 이때, 비밀번호가 일치하지 않을 경우 '비밀번호가 틀렸습니다. 다시 입력해주세요.'라는 에러를 출력합니다. 새 비밀번호는 '새로운 비밀번호를 입력하세요.' 와 '비밀번호를 한번 더 입력하세요.'항목에서 총 두번 입력합니다. 이때, 두 비밀번호가 일치하지 않을 경우, '입력한 두 비밀번호가 다릅니다. 다시 입력해주세요.'라는 에러를 출력합니다.
    ❍ 로그아웃: 서비스에서 로그아웃합니다.
    ❍ 계정 삭제: 비밀번호를 입력하고 계정을 삭제할 수 있습니다. 해당 행동은 복구할 수 없습니다. 이때, 비밀번호가 일치하지 않을 경우 '비밀번호가 틀렸습니다. 다시 입력해주세요.'라는 에러를 출력합니다.

  - 내가 공유 중인 파티: '공유 중인 플랫폼 계정 정보'를 출력합니다. 공유 중인 플랫폼이 없다면 '아직 공유 중인 플랫폼이 없습니다. 플랫폼을 공유해 구독비를 절약해보세요.' 라는 메시지를 출력합니다.
    ❍ 내 파티 구독료 조회: 내가 리더인 파티와 내가 멤버인 파티의 구독료를 조회할 수 있습니다. 소속된 파티가 없을 경우, '아직 내 파티가 없습니다. 새로운 구독을 시작하고 파티에 참여해보세요.'라는 메시지를 출력합니다. 파티에 리더로만 참여할 경우, '아직 멤버로 참여 중인 파티가 없습니다.'라는 메시지가 출력되고 내가 리더인 파티의 구독료만 출력합니다. 파티에 멤버로만 참여할 경우, '아직 리더로 참여 중인 파티가 없습니다.'라는 메시지를 출력하고 내가 멤버인 파티의 구독료만 출력합니다.
    ❍ 구독 중인 플랫폼 조회: 현재 구독중인 플랫폼들을 출력하고, 구독중인 플랫폼이 하나도 없다면 '아직 구독 중인 플랫폼이 없습니다. 새로운 구독을 시작해보세요.'라는 메시지를 출력합니다.
    ❍ 공유 중인 플랫폼 계정 정보 조회: 공유중인 플랫폼의 계정 정보를 출력합니다.

  - 영상 컨텐츠 검색
    ❍ 플랫폼의 모든 컨텐츠 조회: 플랫폼을 선택하면 해당 플랫폼에 존재하는 모든 컨텐츠를 테이블로 출력합니다. 플랫폼은 Disney+, Netflix, Tving, Watcha, Wavve 중 선택 가능합니다.
    ❍ 플랫폼과 평점으로 검색: 플랫폼은 Disney+, Netflix, Tving, Watcha, Wavve 중 선택 가능합니다.  평점은 1에서 5점까지 선택 가능하며, 입력한 평점 이상이고 선택한 플랫폼에 있는 컨텐츠를 검색합니다.
    ❍ 장르와 평점으로 검색: 장르는 Horror, Romcom, SF, Disaster, Blockbuster, Comedy, Action, Animation, Documentary, Classic, Romance, Drama, Fantasy 중에서 선택 가능합니다. 평점은 1에서 5점까지 선택 가능하며, 입력한 평점 이상인 해당 장르의 컨텐츠가 검색됩니다.

  - e-book 검색
    ❍ 작가 직업으로 검색: 작가의 직업을 입력하면 입력한 직업을 가진 작가가 집필한 책을 검색합니다. 해당 직업을 가진 작가가 없다면 '검색 결과가 없습니다. 확인 후 다시 검색해주시기 바랍니다.'라는 에러메시지를 출력합니다.
    ❍ 출판연도 순으로 조회: e-book을 출판연도 순으로 오름차순 정리합니다.

  - 플랫폼 검색
    ❍ 인기 순 조회: Knuflix에서 공유하고 있는 파티가 많은 순으로 조회합니다. 
    ❍ 플랫폼별 비교 조회: 선택한 플랫폼보다 비싼 가격 정책을 가진 플랫폼을 검색합니다. 플랫폼은 Disney+, Netflix, Tving, Watcha, Wavve 중 선택 가능합니다. 선택한 플랫폼보다 비싼 가격 정책을 가진 플랫폼이 없을 경우, '보다 가격 정책이 비싼 플랫폼이 없습니다.'라는 에러를 출력합니다.
  
3. 유의 사항
 - 회원가입 단계에서 사용자 동시성 제어를 위해 이미 존재하는 id로 회원가입을 진행하려 하면 회원가입이 불가합니다.
 - 회원가입 단계에서 사용자 동시성 제어를 위해 이미 존재하는 전화번호로 회원가입을 진행하려 하면 회원가입이 불가합니다.

4. SQL 쿼리 변경사항
 1. Type 9의 '공유하는 파티의 수가 많은 순으로 플랫폼 조회' 쿼리에 구독료를 추가했습니다.
  SELECT a.pltf_name AS Platform, COUNT() AS NUM, f.sub_price
  FROM PARTY a, PLATFORM f
  WHERE a.pltf_name = f.pltf_name
  GROUP BY a.pltf_name, f.sub_price
  ORDER BY COUNT() desc;

 2. Type 6의 'user id로 구독 중인 플랫폼 조회'쿼리에 구독중인 플랫폼 조회시 구독료도 함께 조회되도록 변경하였습니다.
  SELECT  Pltf_name AS Platform, Sub_price
  FROM    PARTY NATURAL JOIN PLATFORM
  WHERE   leader_id = 'ZSnhmzSD0828'
     OR   party_id IN (SELECT party_id
                       FROM PARTICIPATE
                       WHERE user_id = 'ZSnhmzSD0828');
